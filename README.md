# Chat Visual Novel Engine

**Chat VN Engine** — движок для создания визуальных новелл в формате мессенджера, разработанный на **React**, **TypeScript** и **Vite**.

Движок предоставляет готовую систему для создания сюжетов с персонажами, эмоциями, выборами, событиями, загадками и сохранением прогресса.

## Возможности

* отображение диалогов в формате чата;
* поддержка нескольких персонажей;
* смена эмоций персонажей;
* анимация печати сообщений;
* система выборов;
* система событий;
* поддержка загадок;
* система флагов;
* поддержка кампаний, состоящих из нескольких глав;
* автоматическое сохранение прогресса.

---

# Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/iwumekiriko/Chat-VN-Engine.git
```

Установите зависимости:

```bash
npm install
```

Запустите проект:

```bash
npm run dev
```

Для создания production-сборки:

```bash
npm run build
```

---

# Создание собственной новеллы

Весь игровой контент располагается в папке [scenario](https://github.com/iwumekiriko/Chat-VN-Engine/tree/main/src/scenario).

---

## Главы

Каждая глава представляет собой отдельный сценарий диалога.

Диалог строится из массива узлов (`DialogueNode`).

Все доступные значения `DialogueNodeType`, их параметры и примеры использования подробно описаны непосредственно в файлах сценариев.

---

## Персонажи

Персонажи создаются в файле [characters.ts](https://github.com/iwumekiriko/Chat-VN-Engine/blob/main/src/scenario/characters.ts)

Для каждого персонажа можно определить:

* идентификатор;
* имя;
* аватар;
* перечень эмоций.

---

## Кампания

Файл [campaign.ts](https://github.com/iwumekiriko/Chat-VN-Engine/blob/main/src/scenario/campaign.ts) помогает работать с загруженными главами сценария.

---

## Загадки

Все загадки описываются в [puzzles.ts](https://github.com/iwumekiriko/Chat-VN-Engine/blob/main/src/scenario/puzzles.ts).

После успешного решения загадки установливается необходимый флаг и продолжается выполнение сценария.

---

## Флаги

Флаги используются для хранения состояния прохождения.

С их помощью можно:

* проверять выполнение условий;
* управлять логикой сценария.

---

# Сохранение прогресса

Движок автоматически сохраняет состояние прохождения в Local Storage браузера.

При повторном открытии страницы прогресс восстанавливается автоматически.
